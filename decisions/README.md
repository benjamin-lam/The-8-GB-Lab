# Decisions (ADR)

Dieses Verzeichnis enthält Architecture Decision Records (ADRs).

ADRs sind kurze, versionierte Entscheidungen.
Sie sind nicht der Worklog.

- Worklog = Chronologie (was ist passiert, warum, was als nächstes)
- Decisions = Kanon (welche Entscheidung gilt, inkl. Kontext und Konsequenzen)

## Regeln

- Eine Datei pro Entscheidung.
- Nummerierung ist fortlaufend und stabil.
- Dateiname: `NNNN-kebab-case.md`

Beispiel:
- `0003-static-site-pipeline.md`

## Status

Jede Entscheidung hat einen Status:

- **proposed** (Vorschlag, noch nicht final)
- **accepted** (gilt)
- **superseded** (ersetzt durch eine neuere ADR)
- **rejected** (bewusst verworfen)
