# Docs

Diese Dokumentation ist **ein laufendes Log** – kein fertiges Buch.
Ziel: Denken, Entscheidungen, Experimente und Metriken so festhalten, dass das System nachvollziehbar wird.

**Regel:** Jede Änderung am System bekommt eine kurze Doku-Spur:
- 1 Experiment-Log (Hypothese → Änderung → Messung → Ergebnis)
- optional 1 ADR (Architecture Decision Record) für Entscheidungen, die später schwer zu revidieren wären

Einstieg:
- `00_overview.md`
- `10_meta_workflow.md`
- `20_constraints.md`
- `30_metrics.md`
- `40_experiments/`
- `90_decisions/`
