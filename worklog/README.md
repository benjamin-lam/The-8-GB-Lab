# Worklog

Dieses Verzeichnis dokumentiert die laufende Entwicklung des Projekts.

Der Worklog ist kein Tagebuch und kein Marketingtext.
Er ist ein technisches Protokoll für Entscheidungen, Experimente und Richtungsänderungen.

Ziel:
Nachvollziehbar machen

- warum Entscheidungen getroffen wurden
- welche Annahmen existierten
- welche Experimente funktioniert haben
- welche verworfen wurden

## Format

Eine Datei pro Tag.

YYYY-MM-DD.md

Jeder Eintrag folgt grob dieser Struktur:

Context  
Was war der Ausgangspunkt?

Decision  
Welche Entscheidung wurde getroffen?

Why  
Warum wurde sie getroffen?

Result  
Was ist aktuell der Stand?

Next  
Was ist der nächste logische Schritt?
